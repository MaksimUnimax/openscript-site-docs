## What it says in practice

- choose a bold aesthetic direction before building
- avoid generic AI-style UI defaults
- commit to typography, color, spacing, background, and motion as one coherent system
- implement real working UI, not just a moodboard

## Why it may help OpenScript

- good baseline for making lesson pages feel intentional and non-generic
- helpful for creating a stronger visual shell for interactive lessons

## Risks

- if used alone, it may become style-first without enough QA or accessibility review
