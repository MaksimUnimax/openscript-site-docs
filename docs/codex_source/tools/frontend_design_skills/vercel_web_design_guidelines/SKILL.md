## What it says in practice

- review UI against accessibility and interface rules
- use semantic HTML and visible focus states
- check forms, images, motion, typography, and content handling

## Why it may help OpenScript

- useful for validating lesson UI and interactive controls after a creative candidate is chosen
- good for finding accessibility and semantic HTML gaps

## Risks

- review-only; it does not create a distinctive design direction on its own
