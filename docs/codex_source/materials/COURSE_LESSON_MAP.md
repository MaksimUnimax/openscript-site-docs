# Course lesson map

## COURSE_LESSON_MAP_20260529_CURRENT_10_LESSONS

Текущая структура курса:

1. **Как устроена работа с ИИ-разработкой**  
   Роли и общая схема работы: пользователь, ChatGPT, Codex, отчёт и проверка результата.

2. **Начало работы с ChatGPT**  
   Старт проекта через обсуждение с ChatGPT. ChatGPT выступает как профессиональный архитектор и ведущий программист, уточняет цель, предлагает архитектуру, стек, roadmap, ТЗ и готовит задачи для Codex.

3. **Документация как контроль разработки**  
   Документация - не только память проекта. Она контролирует разработку по ТЗ и roadmap, не даёт уйти в сторону и помогает оценивать результат Codex.

4. **Обновление документации**  
   Когда и как обновлять документацию после важных решений, изменения правил, принятого результата и изменения статуса.

5. **Новый диалог через документы**  
   Как безопасно начинать новый диалог через актуальные документы, а не по памяти. Контекст, документы и prefix-правила помогают не потерять состояние проекта.

6. **Git: история, commit, push и откат**  
   Git на уровне владельца проекта: репозиторий, commit, push, diff, откат и объяснение deploy key для учеников.

7. **Как проходит рабочий run Codex**  
   Один безопасный run Codex: proof, одна задача, проверки, commit/push, отчёт, проверка ChatGPT и приёмка пользователем.

8. **PowerShell, Terminal и подключение к серверу**  
   Базовые команды терминала и SSH, подключение к серверу, безопасное выполнение команд, git status/log/diff.

9. **Сервер, Codex, AGENTS.md и Skills**  
   Серверная среда, авторизация Codex, permissions, выбор модели, AGENTS.md и Skills.

10. **Ошибки, лайфхаки и правила работы**  
   Частые ошибки и практики: редкая проверка результата, слишком большие runs, обновление документации, новости ИИ-инструментов, готовые open-source решения, Chrome-расширение для напоминания правил, контекстное окно и запрет длинного диалога с Codex.

### Требования к поведению курса

- В навигации видно 10 уроков.
- URL поддерживает `?lesson=lesson-1` ... `?lesson=lesson-10`.
- Не использовать `location.hash`.
- Не использовать `scrollIntoView`.
- Все видимые labels на русском.
- Прогресс теста считается по отвеченным вопросам, а не только по правильным ответам.
- Неправильный ответ тоже засчитывается как выполненный вопрос.
- В курсе не должно быть старого примера “Сделай мне ИИ-агента”, если он явно не возвращён отдельным решением.
- Не должно быть видимого текста “Тестовая версия курса”.
- Не должно быть textarea-практики.
