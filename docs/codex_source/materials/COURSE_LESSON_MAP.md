# COURSE LESSON MAP

Draft map for the first course inside "Работа с ИИ".

Lesson 1 - What we build and how ChatGPT + Codex work together
- goal: understand the full workflow
- outcome: user can explain the role of docs, prompts, proof, and the app repo

Lesson 2 - Server and project structure in simple words
- goal: understand where the website, docs, and runtime fit
- outcome: user can tell docs repo from app repo from runtime

Lesson 3 - Git and source-of-truth basics
- goal: understand tracked source, commit, branch, and remote
- outcome: user can read a basic git status and know what matters

Lesson 4 - Reading reports and checking proof
- goal: understand the report contract and validation style
- outcome: user can spot what was proven and what was not

Lesson 5 - Small safe changes
- goal: understand how to keep scope narrow
- outcome: user can describe a safe minimal change

Lesson 6 - Tests and acceptance
- goal: understand why tests prove behavior
- outcome: user can read targeted and full test results

Lesson 7 - Git deploy key and GitHub push, explained simply
- goal: understand the release path at a high level
- outcome: user knows why tracked source is the accepted output

Lesson 8 - First real mini-feature
- goal: apply the workflow to a tiny feature
- outcome: user can follow the full loop once end to end

Status:
- draft
- can be revised after tooling and pipeline design are finalized
