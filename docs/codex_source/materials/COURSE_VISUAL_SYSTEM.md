# COURSE VISUAL SYSTEM

The course should feel visual and readable, not like a text wall.

Preferred building blocks:
- cards
- step lists
- diagrams
- code and report callouts
- progress markers
- side notes and checklists

Layout direction:
- simple first
- readable on desktop and mobile
- short sections with clear hierarchy
- course content should be easy to scan before it is easy to deep-read

Motion and media:
- later lessons may use animation or video snippets
- HyperFrames is a later reference for those richer visual pieces
- do not depend on video for the first usable course shell

Design rule:
- clarity first, decoration second
- do not turn the course into a dense landing page or a generic dashboard
- every visual choice should help the learner understand the workflow faster
