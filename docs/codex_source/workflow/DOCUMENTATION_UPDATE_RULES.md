# Documentation Update Rules

Documentation is project memory.

Update docs after:
- new product decision;
- changed roadmap;
- accepted/rejected landing structure;
- accepted/rejected course/materials structure;
- important Codex run;
- new tool adopted;
- new constraint added;
- new open question appears.

Recommended files to update:
- CURRENT_STATUS.md
- CONTEXT.md
- DECISIONS.md
- ROADMAP.md
- MODULE_MAP.md
- HOMEPAGE_SPEC.md
- MATERIALS_SPEC.md
- COURSE_METHOD_SPEC.md
- TOOLS_INDEX.md

Do not update docs with:
- secrets;
- raw tokens;
- temporary logs;
- runtime-only state;
- unverified claims.
