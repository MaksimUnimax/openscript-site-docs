# User Flows

## Before payment

1. User opens landing.
2. User registers.
3. User verifies email.
4. User logs in.
5. User sees cabinet with access not activated.
6. User sees starter tariff/options information.

## After first successful payment

1. User gets permanent materials access.
2. User opens “Работа с ИИ”.
3. User sees materials/courses.
4. User sees environment options and expiration statuses.
5. User can extend options in future payment stage.

## Materials access rule

Materials access remains even if environment options expire.
